语言模型
语言模型发展的四个阶段：N-gram(统计语言模型)、RNN/LSTM（神经网络语言模型）、BERT（预训练语言模型）、LLM（大语言模型）
语言模型：对文字序列赋予概率的数学模型，建模语言中词序列的合理程度，也叫成句概率
N-gram(统计语言模型)、用到16、17年，在语音识别用到比较多
链式规则展开：
$$\mathrm{P}(\mathrm{w_1w_2...w_n}) = \sum_{i=1}^n \mathrm{P}(\mathrm{w_i|w_1w_2...w_i-1})$$
马尔可夫假设（看前N-1个词）：
历史上下文太长难以统计。近似：只看N-1个词作为条件（N取值一般为3~5）
$$\mathrm{P}(\mathrm{w_1w_2...w_n}) = \sum_{i=1}^n \mathrm{P}(\mathrm{w_i|w_1w_2...w_i-1})$$
N-gram频率估计
$$\mathrm{C}(\mathrm{w_i|c_{tx}}) = \frac{\mathrm{C(w_i|c_{tx})}}{C(c_{tx})}$$
零概率问题
训练集中未出现的词组合，其计数 C = 0，导致整个句子概率为零：
拉普拉斯平滑（分子+1分母+词表大小 解决零概率问题）
$$\mathrm{C}(\mathrm{w_i|c_{tx}}) = \frac{\mathrm{C(w_i|c_{tx})}+1}{C(c_{tx})+V}$$
原理如此所以 模型在训练数据里见过的数据概率会更高，训练数据里没见过的数据概率更低
N-gram局限性
- 数据稀疏
  - N 越大，词组合呈指数级增长
  - 训练集难以覆盖所有词组合
  - 大量 N-gram 的出现概率为零   softmax解决
- 长距离依赖失效                          不做物理切割
  - 仅记忆前 N-1 个词，窗口固定
  - 例：「那只在河边散步的 ___」
  - 主谓跨度超过 N 时无法建模
- 语义泛化弱                               embeding解决
  - 基于字符串匹配，无语义理解
  - 「猫吃鱼」≠「猫咪吃鱼」（概率无关）
  - 同义词无法共享统计信息
- 存储开销巨大                             模型大小只与模型权重参数有关，与语料大小无关
  - 存储量 O(Vᴺ)，随 V 指数增长
  - V=50,000 时 3-gram 约 125 亿条
  - 大 N 时内存/磁盘无法承载

循环神经网络语言模型瓶颈
- 顺序计算，无法并行                          attention解决
  - RNN/LSTM 按时间步串行计算
  - 长序列训练时间极长
  - GPU 并行加速无法充分利用
- 仍有记忆衰减                                 
  - LSTM 改善了但未彻底解决
  - 超长文档（>1000词）效果下降
  - 书籍级别语境无法有效利用
- 迁移能力有限                                  预训练 + 微调范式
  - 每个下游任务需从头训练模型
  - 需要大量有标注训练数据
  - 不同任务间知识无法复用
- 参数利用率低
  - 模型容量受单向序列结构限制
  - 无法同时利用左右两侧上下文
  - 双向 LSTM 是折衷方案，代价高
BERT（预训练语言模型）
NLP困境（bert之前）
- 每个任务从头训练，依赖大量标注数据
- 词向量静态，无法感知上下文语义变化
- 任务间知识无法复用
bert使用预训练 + 微调范式
预训练任务一：Masked Language Model
随机遮盖输入序列中 15% 的 Token，让模型预测被遮盖的原始词.
预训练任务二：Next Sentence Prediction
给定两个句子，判断 B 是否是 A 的下一句。训练时 50% 正例（真实相邻句），50% 负例.
bert同时优化两个训练目标 Loss = MLMLoss + NSPLoss
